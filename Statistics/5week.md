# 1️⃣ 개념 정리 
## 01. 회귀모형의 형태 바꾸기

```
✅ 학습 목표 :
* 이원배치 분산분석에 대해 설명할 수 있다.
* 비선형회귀에 대해 설명할 수 있다.
```
### 상호작용
: 설명변수 간의 상승효과, 선형회귀모형 안에서 곱셈 $c*x_iy_j$ 

**< 상호작용항 적용하면 좋은 경우 >**  
	⁃	상호작용이 있다는 것이 선행 연구에서 밝혀지거나 기대되는 때  
	⁃	데이터에 분명한 상호작용이 있을 때  
	⁃	상호작용 유무에 관심이 있을 때

### 이원배치 분산분석
`다원배치 분산분석` : 여러 개의 요인을 동시에 고려  
**`이원배치 분산분석`** : 2개의 요인

가설검정 결과 상호작용항 $c_1$ 이 유의미하지 않다면 상호작용이 없다고 보고, 각각의 **주효과(main effect)** 를 그대로 평가

### 비선형회귀

선형모형 => 파라미터에 관해 선형이라는 뜻

어느 모형이 적절한가, 애당초 ’적절’이란 말은 어떤 관점에서 본 적절함인가를 생각해야 함

`미하엘리스-멘텐식`



## 02. 일반화선형모형의 개념

```
✅ 학습 목표 :
* 선형회귀가 적절하지 않은 상황을 판단할 수 있다.
* 가능도와 최대가능도 방법을 이해한다.
* 로지스틱 회귀와 오즈비에 대해 이해한다.
* 푸아송 분포와 푸아송 회귀에 대해 이해한다.
* 다양한 일반화선형모형에 대해 설명할 수 있다.
```

### 선형회귀 원리 확장하기

**`일반선형모형 (general linear model)`** : 설명변수가 양적변수인 다중회귀부터 설명변수가 범주형 변수인 분산분석까지를 포괄

**`일반화선형모형(GLM, Generalized Linear Model)`** : 최소제곱법이 아닌 확률분포에 기반한 최대가능도방법(최대우도법)으로 회귀모형을 추정

데이터 성질을 고려하면서 확률 모형을 가정하고(베이즈 추정이라면 사전분포도 설정하고), 파라미터를 추정하여 모형을 평가하는 일련의 작업을 가리켜 **통계 모형화**라 부름

### 가능도와 최대가능도 방법
‘확률적으로 얼마나 나타내기 쉬운가‘에 기반해 데이터에 잘 들어맞는지 평가

얻은 데이터를 $x$ ($x_1, x_2, …, x_n$) 로 
확률분포 파라미터를 $\theta$ 로 표기하면,
데이터 $x$ 가 특정 파라미터 $\theta$ 의 확률분포에서 어느 정도 나타나는지를 다음과 같이 쓸 수 있음

P ( x | $\theta$ ) = $ P ( x_1 | \theta ) x … x P ( x_n | \theta) $

**`가능도(우도, likelihood)`** : P(x | \theta)를 데이터 x에 대한 파라미터 $\theta$의 함수로 본 것 

$L(\theta | x) = P(x | \theta)$

**`최대가능도 방법(maximum likelihood method)`** : 가능도를 최대화하는 $\theta$ 를 찾아서, 이를 추정값으로 삼으면 가장 잘 들어맞는 파라미터 $\theta$ 를 정할 수 있음

### 로지스틱 회귀(logistic regression)
: 종속변수가 값이 2게인 범주형 변수일 때 사용  
ex. 종속변수가 예/아니오, 병에 걸림/안걸f림, 생존/사망

`p` : 범주 하나가 일어날 확률  
`p - 1` : 다른 하나가 일어날 확률

독립변수 x 가 바뀌었을 때, p 가 얼마나 달라지는지를 조사

**[ 이항분포 ]**
: 로지스틱 회귀에 사용한 이산형 확률분포 / B(N, p)
$$P(x; N) = {}_N C_x p^x (1 - p)^{N-x}$$

**[ 로지스틱 함수 ]**

**로짓 함수(logit function)** 

$$a + bx = \log \frac{p}{1 - p}$$

일반화선형모형에서는 좌변 a+bx 부분을 ***선형예측변수***라 함.
선형예측변수와 반응변수의 확률분포의 파라미터를 잇는 함수를 ***연결함수***라 함

**[ 오즈비(OR, odds ratio) ]**
: 로지스틱 회귀의 결과

**`오즈(승산, odds)`** : 어떤 사건이 일어날 확률 `p`와 일어나지 않을 확률 `1-p` 의 비율

**`오즈비`** : 2개의 확률 p와 q에 대한 2개의 오즈 비율  
$$OR = (p / (1-p)) / (q / (1-q))$$

### 푸아송 회귀
: 데이터가 음수가 되지 않는 정수일 때, 특히 반응변수가 개수인 경우 고려해볼 수 있는 일반화선형모형

**푸아송 분포** : 낮은 확률로 일어나는 무작위 사건에 대해, 평균이 $\lambda$ 번 일 때 몇번 일어나는지를 나타내는 확률분포

### 다양한 일반화선형모형

`음이항 회귀` : 음이항 분포를 오차분포로 함  
`감마 회귀` : 감마 분포를 오차분포로 함

**[ 과분산 ]**
: 분포에서 규정된 평균과 분산의 관계보다도 분산이 큰 경우

**[ 일반화선형혼합모형 ]**
: 일반화선형모형에 임의효과를 적용한 모형

고정효과(fixed effect) : a 나 b 나 모든 x 에 대해 공통   
임의효과(random effect) : $r_i$ 는 다름






## 03. 통계 모형의 평가와 비교

```
✅ 학습 목표 :
* 왈드 검정과 가능도비 검정에 대해 설명할 수 있다.
* AIC와 BIC에 대해 설명할 수 있다.
```

### 왈드 검정(Wald test)
최대 가능도 방법으로 얻은 추정값/표준오차를 **왈드 통계량**이라 함  
: 왈드 통계량으로 이용하여 신뢰구간이 p 값을 얻는 검정 방법

### 가능도비 검정
: 비교할 2개의 모형 중 어느 한쪽이 다른 한쪽을 포함하는 관계여야 함

**부트스트랩 방법** : 어떤 가정하에 무작위로 데이터를 생성하고 추정량의 성질을 조사하는 방법

### AIC (아카이케 정보기준, Akaike information criterion)
: 새롭게 얻을 데이터를 얼마나 잘 예측할 수 있는지를 바탕으로 모형의 좋음(적합도)를 결정하는 지표

**과대적합** : 실제 데이터에 무리하게 맞춰, 새롭게 얻은 데이터는 제대로 나타내지 못하는 상태

> 주의사항
>> : 새롭게 얻을 데이터의 예측도를 높이는 모형을 고르는 것이 목적인 지표이므로, AIC 를 최소화한다고 해서 그것이 반드시 실제 모형(데이터를 생성한 분포)이지는 않을 수도 있다 !

### BIC (베이즈 정보기준, Bayesian information criterion)
$$ BIC = -2logL* + klog(n) $$

	< AIC 와의 차이점 >
	•	: 표본크기 n 에 따라 달라짐
	•	: 표본크기 n 이 클수록 파라미터 개수 k의 패널티가 커짐


## 04. 재현성

```
✅ 학습 목표 :
* 재현성 위기에 대해 설명할 수 있다.
* 재현 불가능한 원인을 설명할 수 있다.
```

### 재현성 위기
: 재현성이 없는 것 ( 즉, 원래 논문의 주장이 잘못되었을 가능성이 있다는 것 )

**재현성(reproducibility, replication)**
: 누가 언제 어디서 실험하더라도, 조건이 동일하다면 동일한 결과를 얻을 수 있어야 함

### 재현 불가능한 원인

	⁃	실험 조건을 동일하게 조정하기 어려움
	⁃	가설검정의 사용 방법에 따라 p 값이 0.05보다 작아지게 조작하는 것이 가능함 ( p-해킹 )

가설검정 원리에서는 `제1종 오류`를 유의수준 a 로 설정하고, 분석자가 이를 통제할 수 있다는 점이 가장 중요



## 05. 가설검정의 문제점

```
✅ 학습 목표 :
* 왜 유의수준 α = 0.05를 사용하는지 설명할 수 있다.
* 피셔류 검정과 네이만-피어슨류 검정 체계의 이론적 차이와 적용 목적을 설명할 수 있다.
* 효과크기를 이해한다.
* 베이즈 인수를 이해한다.
```

### 왜 유의수준 a=0.05를 사용하는가?
**근거는 없음**  
`로널드 피셔`가 자의적임을 인정하면서도 증거의 강력함을 나타내는 지표로 사용한 것이 그 시작. 현대과학에서는 이를 ***관례***처럼 사용함.

a=0.05 라는 숫자를 사용하여 얻은 통계적으로 유의미한 결과 중, 실제로는 귀무가설이 옳았을 비율이 높기도 함. (즉, 오류라는 것)

### 피셔류 검정과 네이만-피어슨류 검정
**`피셔류 검정`** : 귀무가설이 옳을 때 관찰한 데이터 이상으로 극단적인 값을 얻을 확률인 p 값을 계산하고, 귀무가설과 관찰한 값의 괴리 정도를 평가

**`네이만-피어슨류 검정`** : p 값이 유의수준 a 미만인가 이상인가에만 주목하여, 가설 기각/채택의 결론을 내림

> 단, 미리 검출하고자 하는 효과크기를 정하고, 설정한 $\alpha$ 와 $\beta$에 따라 필요한 표본크기 n 을 결정해야 함

### 효과크기(effect size)
: 얼마만큼 효과가 있는지를 나타냄

**`Cohen's d`** : 원래 모집단의 분산을 기준으로, 2개의 모집단평균이 얼마나 떨어져있는지를 나타냄

**< 다양한 효과크기 >**

| 종류 | 지표 | 용도|  
|:-----|----  |-----|  
| **집단 간 비교** | **Cohen's $d$** | 두 집단의 평균 차이를 표준편차 단위로 나타냄 |
| **상관관계** | **Pearson's $r$** | 두 변수 사이의 선형적 관계의 강도 |
| **분산 분석(ANOVA)** | **$\eta^2$ (Eta-squared)** | 전체 변동 중 특정 요인이 설명하는 비율 |
| **회귀 분석** | **$R^2$ (결정계수)** | 모델이 데이터를 얼마나 잘 설명하는지 |

### 베이즈 인수 (Bayes factor)

p 값은 귀무가설을 기각하는 것만 알 수 있고, 채택할 순 없음  
이를 해결하는 것이 **베이즈 인수** 임

`주변 가능도`  
`에비던스`

$$p(x|M) = \int p(x|\theta, M) p(\theta|M) d\theta$$

**< 베이즈 인수의 특징>**

```
	⁃	2가지 가설을 대등하게 비교 가능. (귀무가설 지지 가능)
	⁃	베이즈 인수가 특정값이 될 때까지, 표본크기 n 을 늘리는 순차적인 갱신 가능
	⁃	데이터를 예측하는 정도가 2개 모형에서 동일할 때, 파라미터가 “적은” 단순모형이 선택 됨
	⁃	비교하는 모형 중에 실제 모형이 포함되어 있다면, 표본크기 n -> 무한대일 때, 실제 모형을 고를 확률이 “1에 한 없이 가까워지는 성질”이 있음
```

**< 베이즈 인수의 주의점 >**
```
	⁃	두 가설의 상대적인 비교일 뿐이어서, 한 쪽 가설이 나쁜 것만으로 베이즈 인수가 큰 값이 될 수 있음 (따라서 사후예측점검 필요)
	⁃	베이즈 인수는 파라미터 세타의 사전분포에 영향을 받음
	⁃	주변 가능도를 구할 때는 모형으로 설정한 파라미터로 평균화하기 위한 적분 계산이 필요함 (따라서 시간, 노력이 많이 듦)
```

**허위발견율** : 옳다고 주장된 것 중 위양성인 것의 비율





## 06. p-해킹

```
✅ 학습 목표 :
* p-해킹의 개념과 특징을 설명할 수 있다.
```

### p - 해킹 (p-hacking)
: 의도하든 의도하지 않든 p값을 원하는 방향(유의수준 a=0.05 미만이 되도록)으로 조작하는 행위

>< 구체적인 예 >
>> 1) p < 0.05 가 될 때까지 표본크기 n을 늘림
>> 2) 처음에는 n=30으로 실험하여 p=0.07이었지만, 표본크기 n=10을 추가하여 n=40으로 실험했더니, p < 0.05 이 되었기에 이를 보고함
>> 3) 여러 개의 요인을 탐색하여, 그 중  p < 0.05 인 것만 보고함

<br>

- 결과를 보며 표본크기를 늘려서는 안됨.
- 마음에 드는 해석만 보고해서는 안됨. 
   * **HARKing** : 데이터를 얻어 결과를 보고 나서 가설을 만드는 행위

### p-해킹을 예방하기 위한 노력들

**`탐색형 연구`** : 전체를 탐색적으로 해석하는 연구  
**`가설검증형 연구`** : 가설을 세우고 이를 검증하는 연구

**사전등록** : 연구를 실시하기 전에 가설과 실험 설계, 분석 방법 등의 연구 계획을 등록하는 것
$\rightarrow$ HARKing 을 막을 수 있음

### 가설검정을 이해할 때 확인할 항목

- p값이란 귀무가설이 옳을 확률을 말한다 (x)
- p값이란 우연만으로 관찰 데이터가 나타날 확률이다 (x)
- 유의미한 결과(p<0.05) 란 귀무가설이 틀리다는 뜻이다 (x)
- p값이 클수록 귀무가설을 지지한다는 증거이다 (x)

## 문제 1.

> **🧚Q. 한 연구자가 신약의 효과를 검증하기 위해 실험을 진행하였다.
  처음 분석에서는 p = 0.12로 유의하지 않았다.  
  그러나 이후 일부 이상치를 제거하고, 설명변수를 추가한 뒤 다시 분석한 결과 p = 0.03이 나왔다.  
  연구자는 최종 결과(p = 0.03)만을 논문에 보고하며  
  “신약은 통계적으로 유의한 효과가 있다”고 결론 내렸다.**

> **다음 중 위 상황을 가장 적절하게 설명한 것은 무엇인지 고르고 선택한 답의 이유를 간단히 서술하시오.**

~~~
1️⃣ 연구자는 데이터를 정제했으므로 통계적으로 더 정확한 결과를 얻은 것이다.  
2️⃣ 분석을 반복했으므로 제2종 오류가 감소한 것이다.  
3️⃣ 유의한 결과만 보고했으므로 p-해킹 및 제1종 오류 증가의 가능성이 있다.  
4️⃣ p값이 0.05보다 작으므로 과학적으로 효과가 증명되었다.
~~~

```
3️⃣ 입니다. 연구자가 유의미한 결과를 얻기 위해 분석방법을 임의로 변경했으며, 마음에 드는 해석만 보고했기 때문에 제 1종 오류의 확률이 커질 수 있음.
```
