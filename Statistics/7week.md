### 변수의 차원

“차원이 높다” → 정보량이 많다 → Good? → 쓸데없이 많기만 한 상황 발생

⇒ **차원 축소(dimension reduction)** : 변수의 수를 줄이는 것 필요 !

### 

<aside>
💡 변수의 수를 줄이는 이유

- 고차원 데이터 해석의 어려움
- 다중공선성
- 표본크기 n이 충분하지 않을 때, 회귀계수를 올바르게 추정할 수 없음
</aside>

### 주성분분석(PCA, Principle Component Analysis)

: 차원 축소의 가장 기본적인 방법

`PC1 (제1주성분)` : 데이터 퍼짐이 가장 커지는 방향으로 설정하는 첫번째 축

`PC2(제2주성분)` :  PC1과 수직을 이루면서, 남은 분산을 가장 많이 설명하는 축입니다

⇒ **원래 각 축에 있던 정보를 PC1 으로 압축해 넣는 것**


**기여율** : 각 주성분이 가진 정보(분산)의 비율

**누적기여율** : 제1부터 제k주성분까지 전체 정보의 몇 %가 포함되는지

**주성분부하량(인자부하량)** : 각 주성분이 어떤 축인지를 조사하기 위해 각 주성분의 값과 원래 각 변수의 상관관계를 계산 → 이 값을 일컬음

### 주성분분석 결과 예시

|  | PC1 | PC2 |
| --- | --- | --- |
| 기여율 | 0.68 | 0.26 |
| 누적기여율 | 0.68 | 0.94 |

→ PC2 까지 전체 정보의 94%가 있음

|  | PC1 | PC2 |
| --- | --- | --- |
| 국어 | 0.08 | 0.93 |
| 수학 | 0.96 | -0.22 |
| 과학 | 0.98 | 0.07 |
| 사회 | 0.28 | 0.90 |

→ PC1은 수학, 과학과 강한 양의 상관관계 / PC2는 국어, 사회와 강한 상관관계

→ PC1은 이과 과목, PC2는 문과과목

**주성분점수** : 원래 데이터를 새로운 변수를 이용

| ID | PC1 | PC2 |
| --- | --- | --- |
| 1 | 47 | -32 |
| 2 | 51 | 28 |

→ ID1은 이과 과목은 잘하지만 문과과목은 못함, ID2는 반대

### 인자분석(factor analysis)

원래 각 변수에는 소수의 공통인자가 있으며, 이것이 원인이 되어 데이터의 각 변수가 구성된다는 아이디어에 기반을 둔 방법

>주성분분석 : 각 시험 점수를 ‘이과 과목 성적’ 이라는 변수로 요약

>인자분석 : ‘이과 숙련도’라는 잠재적인 인자와 각 과목의 독립적인 요인이 어떻게 합쳐져 수학 점수나 과학 점수가 구성되는지를 조사

분석 자체는 ***상관관계***에 기반을 두므로, 데이터에서 ***인과관계를 발견할 수 없음***

<aside>
💡 기계학습 분류

- **비지도 학습 (Unsupervised learning)** : 데이터 요약, 특징 추출이 주된 목적
- **지도 학습 (Supervised learning)** : 데이터 간의 관계를 학습

→ 정답 데이터의 유무에 따라 비지도/지도 학습

- **강화학습 (Rainforcement learning)** : 정답 데이터는 주어지지만, 직접적이지 않음. 개체가 환경에 대해 *행동*을 취하고, 이에 따른 *보상*과 *상태*를 통하여 행동을 갱신하는 원리
</aside>

### 통계학과 기계학습의 차이

- **통계학** 은 작은 표본크기를 가진 데이터를 대상으로 하며, 설명이나 해석을 중시
- **기계학습** 은 ***대량***의 데이터를 대상으로 하고, ***예측***을 중시

→ 기계학습 모형은 복잡하고, 모형을 해석하기 어려우나 표현력이 강하고 예측 성능이 높음

<aside>

: 정답 데이터가 없으며, 데이터의 배후에 있는 구조를 올바르게 추출하려는 목적으로 사용 됨

### 군집분석(cluster analysis)

: 각 데이터가 어떤 군집에 속하는지를 구하는 방법

`k-평균(k-means)` 

- 군집 개수 k 를 정함
- 각 데이터에 무작위로 군집을 할당하고, 각 군집의 중심위치(데이터의 평균)를 구함
- 데이터마다 가장 근접한 군집을 할당 → 중심 위치를 다시 계산

`계층적 군집화(hierarchical clustering)`

: 모든 데이터가 다른 군집에 속하는 상태에서 시작하여, 가장 거리가 가까운 군집끼리 순서대로 합쳐 나감

*덴드로그램* 을 그릴 수 있음

- 군집 개수는 미리 정해지지 않음
- 어느 단계에서 바라보느냐에 따라 달라짐

⇒ 군집분석에서는 ***데이터 사이의 거리를 계산*** 
⇒ but, 거리의 정의를 분석자가 지정하기에, ***결과에는 분석자의 자의성이 개입*** 되는 것을 염두

### PCA 이외의 차원 축소 방법

주성분분석은 ***선형 상관관계***에 기반을 두므로, 비선형관계에는 적용 X

`t-SNE(t-distributed Stochastic Neighbor Embedding)`

`UMAP(Uniform Manifold Approximation and Projection`

→ 복잡한 비선형관계에도 적용할 수 있는 방법

### 지도학습이란?

= 교사 있는 학습 ⇒ 설명변수 x에 대응한 정답 y값이 주어지는 것

특정 x 값일 때, y가 어떤 값이 되는지에 관한 데이터를 바탕으로 그 관계를 ***학습***해 나가는(함수를 구하는) 방법

>**학습** 이란? 

>> 모형에서 얻을 수 있는 출력을 가능한 한 실제 y에 가깝도록 하고자 모형의 파라미터를 갱신하는 것

>> 모형의 출력과 실제 y의 차이를 **손실함수**로 정식화, 이를 최소화하는 파라미터를 구함
>> 반응변수 y가 양적 변수일 때는 **회귀**, 질적 변수일 때는 **분류**

### 예측과 교차검증

**예측** : 동일 조건에서 얻을 수 있는 미지의 데이터에 대해 설명변수 x로 반응변수 y를 예측하는 것

**교차검증(cross validation)** : 한쪽 데이터를 사용해 ***학습***을 진행하고, 나머지 한쪽 데이터는 ***예측***이 얼마나 좋은지 평가하는 데 이용

- K-분할 교차검증 : 데이터를 K개로 나눈 뒤 K-1개 학습 데이터, 1개 검증 데이터
- Leave-one-out 교차검증

**과대적합(overfitting)** : 학습 데이터에 대한 예측 성능은 높지만, 검증 데이터에 대한 예측 성능은 낮을 때

⇒ 즉, 학습 데이터의 무작위 데이터 퍼짐에도 모형이 적합해지고 말아, *검증 데이터를 제대로 예측하지 못한다는 것을 나타냄*

**정칙화(regularization)** : 모형의 복잡도를 조정하는 요소를 손실함수에 도입할 때

### 예측 성능 측정 1) 이진 클래스 분류

반응변수 2개의 클래스 A, B일 때를 ***이진 클래스 분류*** 라고 함

<aside>
💡

- 위양성(FP) : 실제는 음성이나, 양성으로 판정
- 위음성(FN) : 실제는 양성이나, 음성으로 판정
- 진양성(TP) : 양성을 양성으로 올바르게 판정
- 진음성(TN) : 음성을 음성으로 올바르게 판정
</aside>

⇒ 진양성, 진음성이 많고 위양성, 위음성이 적을수록 ***예측 성능이 좋은 것***

예측 성능을 숫자로 평가하기 위해, 목적에 따라 다음과 같은 값을 사용

<aside>
💡

- **민감도** : 양성을 양성이라 올바르게 판정할 비율
- **특이도** : 음성을 음성이라 올바르게 판정할 비율
- **양성 적중률** : 양성이라 판정한 것이 실제로 양성일 비율
- **음성 적중률** : 음성이라 판정한 것이 실제로 음성이라 비율
- **정답률** : 모든 판정 중 올바르게 예측한 것의 비율
</aside>

일반적으로 민감도가 높을수록 특이도가 낮다는 *상충관계* 가 있습니다.

**ROC(Receiver Operating Characteristic) 곡선**
: 분류의 예측 성능을 평가

**AUC(Area Under the Curve)**
: ROC 곡선 아랫부분의 넓이 → 1에 가까울수록 예측 성능이 좋음

### 예측 성능 측정 2) 회귀

평균제곱오차

제곱근평균제곱오차

평균절대오차

---

**로지스틱 회귀** : 선형모형이므로, 단순한 분류만 가능한 선형분류기

---

**결정트리(decision tree)**
: 조건 분기로 트리 구조를 만들어 데이터를 분류하는 방법

**랜덤 포레스트(random forest)** 
: 결정트리의 예측 성능을 향상시키기 위해 여러 개의 결정 트리를 구성하고, 이를 다수결로 수행하는 방법

---

**SVM(서포트 벡터 머신)**

: 각 데이터 점과의 거리가 최대가 되는 경계를 구하는 방법
: 일반화 성능이 좋음
: 비선형 분류도 가능함

---

**신경망(딥러닝, neural network)**

: 뇌의 신경 세포(뉴런)의 네트워크를 인공적으로 구성한 모형
: 여러 로지스틱 회귀의 출력이 또 다른 로지스틱 회귀의 입력이 되는 모형

***딥러닝***은 층이 깊은 신경망

***오차역전파법*** : 신경 세포 사이의 가중치를 조정하는 방법

- (+) 신경망은 복잡한 관계를 나타내는 표현력이 높은 방법
- (-) but, 파라미터(가중치) 해석이 어려움
