### 데이터 저장 형태
데이터는 보통 데이터베이스 테이블 등에 저장  
-Database: 데이터의 저장소  
-Table: 데이터가 저장된 공간

데이터는 MySQL, Oracle, PostgreSQL 같은 데이터베이스에 주로 저장됨

### OLTP란?

데이터베이스의 특징 중 하나인 OLTP(Online Transaction Processing).  
- 거래를 하기 위해 사용되는 데이터베이스  
- 데이터가 **무결함**
- 데이터의 추가, 데이터의 변경이 많이 발생
- 서비스에서 많이 사용됨

SQL을 사용해 데이터를 추출할 수 있으나,  
분석을 위해 만든 데이터베이스가 아니라 쿼리 속도가 느릴 수 있음

### SQL(Structured Query Language)
: 데이터 베이스에서 데이터를 가지고 올 때 사용하는 언어  
: 데이터베이스의 데이터를 관리하기 위해 설계된 특수 목적의 프로그래밍 언어  

#### 테이블 구조  
**행(Row)**
- 새로운 row는 가로로 한 줄  
- 하나의 row가 하나의 고유한 데이터  

**열(Column)**  
- 원기둥, 세로
- 각 데이터의 특성 속성 값

### OLAP(Online Analytical Processing)
: OLTP로 데이터 분석 시 속도, 기능 부족 문제로 등장  
: 분석을 위한 기능 제공

*데이터 웨어하우스란?  
: 데이터를 한 곳에 모아서 저장(일종의 창고)  
: 여러 곳에 저장된 데이터 예시  
  :Database, 웹(크롤링), 파일(csv 등), API의 결과 등  

### BigQuery란?
구글 클라우드의 OLAP + 데이터 웨어하우스
### BigQuery 장점
(1) SQL을 사용해 쉽게 데이터 추출 가능  
(2) OLAP 이므로 도속도가 빠름  
(3) 앱, 웹(Firebase, Google Analytics)의 데이터들이 BigQuery에 자동으로 저장되므로 데이터를 쉽게 추출할 수 있음  
(4) 데이터 웨어하우스를 사용하기 위해 서버를 띄울 필요가 없음
### BigQuery 비용
 쿼리비용(SQL) / 저장비용

 ### BigQuery 환경 구성 요소
 (1) 프로젝트(Project): 하나의 큰 건물    
 (2) 데이터셋(Dataset): 프로젝트에 있는 창고  
 (3) 테이블(Table): 창고에 있는 선반  



## 2-1. 데이터 활용 Overview

~~~
✅ 학습 목표 :
* 데이터를 활용하는 과정을 설명할 수 있다.
* 데이터를 탐색하는 과정으로 조건과 추출, 요약을 할 수 있다. 
~~~

### 데이터를 활용하는 과정
task 주어짐 > 원하는 것을 정함 > **데이터 탐색** > **데이터 결과 검증** > 피드백/활용

*문제정의(MECE): 타깃이 누구인지, 지표 정하기, 가설 정하기와 같은 과정이 포함됨
### 데이터 탐색 과정
단일 자료 / 다량의 자료(연결)
- 조건(필터링)
- 추출  ex. 시간 데이터에서 date만 추출
- 변환 ex. 1 -> 1명
- 요약



## 2-2. 저장된 데이터 활용하기

~~~
✅ 학습 목표 :
* 데이터가 저장되는 형태를 알고 저장된 데이터를 활용할 수 있다. 
~~~

#### SQL 쿼리를 작성하기 전에
- 데이터가 어떻게 저장되어 있는가?
- 어떤 데이터가 저장되어 있는가?
- 컬럼의 의미는 무엇인가?

데이터를 제대로 이해해야 올바른 데이터를 추출할 수 있음  
데이터를 추출하기 전에 데이터웨어하우스에 데이터가 어떻게 저장되어 있는지 확인

### 데이터가 저장되는 형태
**ERD(Entity Relationship Diagram** : 데이터베이스 구조를 한눈에 알아보기 위해 사용  
ERD가 없다면 데이터베이스를 직접 탐색 / 스프레드시트나 문서로 따로 정리  
내가 자주 쓰는 테이블부터 정의 > 확장 식으로 정리하기

### 회사에 존재할 수 있는 데이터
(1) 서비스에 사용될 데이터베이스  
(2) 앱/웹 로그 데이터  
(3) 공공데이터, 서드파티 데이터
